# Gestion des champs `NULL` côté backend

Ce guide décrit comment empêcher l’API de renvoyer des chaînes `null` (ex. `reference`, `nom`, `type_paiement`) qui provoquent l’exception Flutter `type 'Null' is not a subtype of type 'String'`.

---

## 1. Base de données

1. **Identifier les colonnes critiques** (collectes.reference, contribuables.nom, collecteurs.prenom, etc.).  
2. **Appliquer des valeurs par défaut** avec `ALTER TABLE` :
   ```sql
   ALTER TABLE collectes
     ALTER COLUMN reference SET DEFAULT CONCAT('REF-', id);
   ```
3. **Nettoyer les enregistrements existants** :
   ```sql
   UPDATE collectes
   SET reference = CONCAT('REF-', id)
   WHERE reference IS NULL;
   ```
4. **Rendre les champs obligatoires** si possible :
   ```sql
   ALTER TABLE contribuables
     ALTER COLUMN nom SET NOT NULL;
   ```

> Astuce : `COALESCE(colonne, 'valeur')` élimine les `NULL` directement dans les vues ou requêtes.

---

## 2. Vues SQL ou requêtes personnalisées

Dans les vues `info_collecte` / requêtes utilisées par FastAPI :

```sql
SELECT
  c.id,
  COALESCE(c.reference, CONCAT('REF-', c.id)) AS reference,
  COALESCE(contrib.nom, 'Contribuable') AS nom_contribuable,
  COALESCE(contrib.prenom, '') AS prenom_contribuable,
  ...
FROM collectes c
LEFT JOIN contribuables contrib ON contrib.id = c.contribuable_id;
```

Ainsi, même si la base contient des `NULL`, la vue renvoie toujours une chaîne valide.

---

## 3. FastAPI / SQLAlchemy

### 3.1. Schémas Pydantic

```python
class CollecteSchema(BaseModel):
    id: int
    reference: constr(strip_whitespace=True) | None = None

    @field_validator("reference", mode="after")
    def default_reference(cls, value, info):
        return value or f"REF-{info.data['id']}"
```

### 3.2. Normalisation dans les endpoints

```python
def normalize_collecte(collecte: dict) -> dict:
    collecte['reference'] = collecte.get('reference') or f"REF-{collecte['id']}"
    collecte['type_paiement'] = collecte.get('type_paiement') or 'inconnu'
    return collecte

@router.get("/collectes")
async def get_collectes(...):
    data = await service.list_collectes(...)
    return [normalize_collecte(c) for c in data]
```

### 3.3. Hooks SQLAlchemy

```python
@event.listens_for(Collecte, "before_insert")
def set_reference(mapper, connection, target):
    if not target.reference:
        target.reference = f"REF-{uuid4()}"
```

---

## 4. Stratégie recommandée

1. Corriger les données existantes (`UPDATE`).
2. Poser des valeurs par défaut en base (assure la cohérence pour tous les consommateurs).
3. Ajouter un filet de sécurité dans le code FastAPI (normalisation + schémas Pydantic).
4. Conserver le parsing robuste côté Flutter pour éviter qu’un oubli backend ne casse la prod.

En combinant ces niveaux, l’API renverra toujours des chaînes valides et l’erreur Flutter disparaîtra définitivement.
